ANNOTATION FIELDS

elapsed_ms
    How many milliseconds have passed since the experiment began on that
    specific machine (Date.now() - experimentWallStartTime).
    Use for sanity-checking timing within a single session only.
    Do NOT use to compare across the two players — each machine has its
    own local clock.

timestamp
    The event time in PupilCore's internal clock (seconds since Pupil
    started). Addded by SendData.py using the paired reference recorded
    at connection. This is the field to use for aligning with gaze data.

duration
    Always 0.0 — annotations mark a point in time, not a span.

added_in_capture
    Always True — annotations are sent live during recording via ZMQ.

label
    The event type: "experiment_start", "phase_start", or "decision".




HOW TO ALIGN GAZE DATA WITH THE EXPERIMENT:

Use phase_start annotations as the main segmentation anchors.
They are the most valid markers for task analysis because they align
to actual on-screen phase transitions.

The experiment_start annotation is still useful as a coarse session-level
reference, but it should NOT be the main anchor for trial/phase analyses.

--- Step 1: Collect phase boundaries ---

    phase_starts = sorted(
        [a for a in annotations if a['label'] == 'phase_start'],
        key=lambda a: a['timestamp']
    )

--- Step 2: Build phase windows ---

    windows = []
    for i, ann in enumerate(phase_starts):
        start_t = ann['timestamp']
        end_t = phase_starts[i + 1]['timestamp'] if i + 1 < len(phase_starts) else None
        windows.append((ann['trial'], ann['phase'], start_t, end_t))

--- Step 3: Segment gaze by trial and phase ---

Each window is [phase_start, next_phase_start).
Use that interval to collect ALL eye data samples that happened while the
phase was actually on screen.

Example — all gaze during trial 5's decision phase:

    phase_ann = next(
        a for a in annotations
        if a['label'] == 'phase_start'
        and a['trial'] == 5
        and a['phase'] == 'decision'
    )

    phase_start = phase_ann['timestamp']
    next_ann = next(
        a for a in phase_starts
        if a['timestamp'] > phase_start
    )
    phase_end = next_ann['timestamp']

    trial5_decision_gaze = [
        g for g in gaze_data
        if phase_start <= g['timestamp'] < phase_end
    ]

Important:
Do NOT just take the first gaze or first fixation after the annotation.
That is not valid alignment. You need the full interval of samples.

For fixations, include any fixation that overlaps the interval, even if the
fixation began before the phase start.

--- Step 4: Align the two players' gaze streams ---

Both players have annotations with the same 'trial' and 'phase' values
but different PupilCore timestamps (each machine has its own clock).
Find the same annotation in each recording to compute the offset:

    offset = player1_ann['timestamp'] - player2_ann['timestamp']
    player2_gaze_aligned = player2_gaze_timestamps + offset

Any matching trial/phase annotation works as the alignment anchor.
You have 242 trials worth of redundant checks.



ANNOTATION TYPES SENT BY THE EXPERIMENT

experiment_start
    Fired once when both players begin the experiment.
    Extra fields: wall_time_ms, event_wall_time_ms

phase_start
    Fired at every phase transition (baseline, sample, delay,
    decision, feedback, postFeedbackDelay).
    Extra fields: trial, phase, elapsed_ms, event_wall_time_ms

decision
    Fired when a player presses a key during the decision phase.
    Extra fields: trial, phase, choice, elapsed_ms, reaction_time_ms,
    event_wall_time_ms
